Typing Ugaritic transliteration on a phone is exactly as painful as it sounds. The standard academic notation uses a small zoo of diacritics — ġ, ḫ, ṯ, ṱ, ʾ, ʿ — none of which appear on any default mobile keyboard. The usual workaround is a clipboard full of copied characters and a lot of patience. I wanted something better, so I built [semitic-keyman](https://github.com/alexsosn/semitic-keyman): a Keyman touch keyboard for Ugaritic transliteration on iOS and Android.

## Why Ugaritic?

Ugaritic is an ancient Semitic language spoken in the city of Ugarit (modern Ras Shamra, Syria) around 1400–1200 BCE. It holds a peculiar place in the history of writing: the script is cuneiform — clay-tablet marks made with a stylus — but unlike the syllabic cuneiform of Akkadian, it is fully alphabetic, with around 30 consonantal signs. It is one of the earliest known alphabets.

Scholars today don't type in actual Ugaritic cuneiform (Unicode has a block for it, U+10380–U+1039F, but it is rarely used in practice). Instead, they use transliteration: a mapping from each Ugaritic sign to a Latin letter, usually with diacritics to distinguish sounds that don't exist in English. Depending on the edition and dictionary you're working from, the conventions differ slightly — which creates its own compatibility headaches.

## What the Keyboard Does

The keyboard is a touch-only layout, which means it is designed from the ground up for mobile screens rather than adapted from a desktop `.kmn` file. It installs in the [Keyman](https://keyman.com/) app on iOS or Android.

**Layers:**

- **Main (ABC):** Standard ASCII letters, plus `ʔ`, `ʕ`, and `ø` as first-class keys — no long-press required for the most common special characters.
- **Shift:** Capital forms, with shifted `ʔ` producing `ʾ` for the aleph marker.
- **Long-press:** Each key exposes transliteration variants, vowel diacritics, and editorial symbols like `⟨/⟩` on long-press. For example, long-pressing `h` gives access to `ḥ`, `ḫ`, and `ḥ̱`.
- **123 layer:** Numbers, punctuation, `√`, and editorial brackets `⸢/⸣` used in critical editions to mark damaged text.
- **Diac layer:** Standalone combining marks with navigation keys, for the times you need to apply a diacritic to an unexpected base character.

## Predictive Text

The more interesting part is the optional dictionary package: `dulat.uga.wordlist.model.kmp`. When installed alongside the keyboard, it provides autocomplete suggestions based on real Ugaritic lexical data drawn from:

- **DULAT** — *Dictionary of the Ugaritic Language in the Alphabetic Tradition* (Olmo Lete & Sanmartín, Brill)
- **UDB** — *Ugaritic Data Bank* (Cunchillos et al.)

Frequency weights come from UDB concordance counts, so common words float to the top of suggestions. The model normalizes across multiple transliteration conventions, so searching for a word in KTU notation will still surface forms that appear elsewhere in a different convention.

Practically, this means the keyboard can predict the next word as you type a Ugaritic text, which is genuinely useful when you're working through a long tablet transcription.

## Technical Implementation

The keyboard source files (`.kmn`, `.keyman-touch-layout`) and lexical model (`.ts`) are generated by Python scripts rather than written by hand. The core of the system is `tools/ugaritic_map.py`, which holds the complete mapping from Ugaritic signs to transliteration characters. Generating from a single source of truth means that the keyboard layout, the dictionary, and any future targets all stay consistent automatically.

Building requires Node.js and the Keyman compiler (`kmc`):

```bash
./build.sh
```

This produces two installable `.kmp` packages in the `build/` directory:
- `ugaritic_translit.kmp` — the keyboard
- `dulat.uga.wordlist.model.kmp` — the autocomplete model

Install by opening the files in the Keyman app, or via **Settings → Install From File**.
